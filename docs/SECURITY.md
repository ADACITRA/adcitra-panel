# AdCtira灯塔 安全策略

## 安全特性

AdCtira灯塔内置多层次安全防护机制，从系统层面到应用层面全面保护您的服务器。

## 安全配置

### 1. 密码策略

在 config/security.json 中配置：

- 最小长度: 8 位
- 必须包含大写字母
- 必须包含小写字母
- 必须包含数字
- 建议包含特殊字符
- 密码有效期: 90 天

### 2. 登录保护

- 失败锁定: 5次失败后锁定30分钟
- Session超时: 60分钟无操作自动退出
- 可启用 2FA 双因素认证

### 3. 防火墙

面板内置防火墙支持：

- IP 黑白名单
- 端口放行/禁止
- CC 攻击防护
- 地区访问限制
- 流量速率限制

### 4. Web 应用防火墙 (WAF)

- SQL 注入防护
- XSS 攻击防护
- 路径穿越防护
- 文件包含防护
- 恶意爬虫识别

### 5. SSH 安全

- 可禁用 root 登录
- 可修改默认 SSH 端口
- 密钥认证支持
- 登录失败自动封禁

### 6. 文件安全

- 文件上传白名单（类型和MIME校验）
- 路径遍历防护
- 敏感文件权限控制
- 木马查杀工具

### 7. 审计日志

- 管理员操作日志
- API 调用日志
- 登录/登出记录
- 安全事件记录

## 最佳实践

### 部署后立即执行

1. 修改默认管理员密码
2. 更改面板默认端口 (8888 -> 其他)
3. 启用 HTTPS 访问面板
4. 配置防火墙只放行必要端口
5. 设置备份策略
6. 配置 Webhook 认证 Token

### 定期维护

- 每周检查安全更新
- 每月审查审计日志
- 每季度更新密码
- 定期备份数据
